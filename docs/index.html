ROI-aware LiDAR compression for detector-facing range images

This project studies whether LiDAR compression can allocate bits by detection relevance instead of treating every spatial region equally. The codebase implements a learned range-image codec, ROI-guided adaptive quantization, dual downstream evaluation tracks, and explicit failure-analysis tooling.

Overview

What the repository actually implements

The active training pipeline uses a 5-channel range image (range, intensity, x, y, z), a resnet or darknet encoder, an ROI importance head, adaptive quantization, and a decoder back to a 5-channel range-image representation.

Stage semantics are explicit in the repo: Stage1 isolates ROI-aware adaptive quantization without distillation, Stage2 adds teacher distillation, and Stage3 is the detector-facing calibration phase that is still being refined.

  • Track 1: point cloud → range image → point cloud, then detector evaluation with PointPillars.
  • Track 2: point cloud → range image, then direct range-image detection with RangeDet.
  • Claim policy: codec diagnostics and detector endpoint metrics are kept separate on purpose.

Pipeline

Two tracks are used to localize where performance breaks

Track 1

Reconstructed point-cloud endpoint

The no-quant chain trains the codec, exports a KITTI_Identity dataset, fine-tunes PointPillars, and evaluates reconstructed point clouds under a detector-facing protocol.

Core script: src/scripts/run_track1_noquant_chain.sh

Track 2

Range-image detector endpoint

Reconstructed range images are exported into a RangeDet-compatible dataset layout, then evaluated directly against a repaired raw/basic detector baseline.

Core script: src/scripts/run_rangedet_stage_eval.sh

Stage logic

Compression objectives are not mixed casually

Uniform, adaptive, distilled, and Stage3 multi-scale variants are tracked as separate stages with explicit metadata, which makes result interpretation much less fragile than raw loss comparison.

Entrypoint: src/main_train.py

Failure Analysis

Three bottlenecks dominate the current story

01

Importance calibration instability

Earlier runs showed empty ROI predictions at a fixed threshold of 0.50, while lower thresholds could recover recall only by introducing very large false-positive regions.

02

Projection-path sparsification before codec comparison

Detector collapse is not explained by under-training alone. A raw vs identity vs reconstructed diagnostic found mean point retention of only 0.4169 after projection and unprojection.

03

Range-image artifact modes are now concrete

The latest Track 2 root-cause cycle points to valid-mask corruption, row-wise banding, and detector-agnostic training objectives as the most credible RI-native failure modes.

Current Direction

What the latest notebook and journal entries say to try next

Pilot A

Mask-aware codec

Prioritize occupancy and valid-mask fidelity instead of treating mask corruption as a side effect.

Pilot B

Anti-banding skip decoder

Attack the row-wise banding signature that now appears repeatedly in Track 2 artifact summaries.

Pilot C

Detector-aware auxiliary target

Feed repaired raw/basic RangeDet teacher maps back into the codec training path.

Pilot D

Combined recovery branch

Merge skip decoding, mask-aware losses, and anti-banding ideas before running a final compare step.
