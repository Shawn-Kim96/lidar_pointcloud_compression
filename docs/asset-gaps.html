Asset wishlist

What would make the page even stronger

The current site is already supportable from local summaries, notebooks, and architecture assets. The remaining gap is mostly visual polish: exported figures and short motion assets that are present in notebook logic but not yet committed as public-facing media.

1. Track 1 qualitative reconstruction panel

Status now: usable Track 1 notebook PNGs have been extracted into docs/assets/, including identity/codec BEV panels and a PointPillars endpoint panel.

Current source: notebooks/track1_identity_vs_codec_visualization.ipynb

Still missing for a final pass: one deliberately selected best-case and one failure-case panel with presentation-level captions, consistent framing, and no notebook UI residue.

2. Track 2 artifact montage for mask corruption and banding

Status now: usable Track 2 PNG exports have been pulled into docs/assets/, so the page is no longer missing Track 2 visuals entirely.

Current source: notebooks/track2_codec_root_cause.executed.ipynb and src/scripts/analyze_track2_codec_root_cause.py

Still missing for a final pass: one deliberately curated “hero” figure with consistent cropping, caption overlays, and a cleaner typography treatment than notebook-default exports.

3. Short GIF of raw/basic versus reconstructed RI evolution

Status now: the site includes lightweight compare GIFs for Track 1 and Track 2, created from still figure panels.

What is still missing: a true temporal RI animation derived from consecutive frames, ideally raw/basic vs reconstructed with synchronized captions and no notebook framing.

Ideal export: GIF or MP4 with synced raw/basic and reconstructed frames plus a tiny caption bar.

4. Raw vs identity vs reconstructed point-retention figure

Why it matters: projection-path sparsification is one of the most important non-obvious findings in the repo, and it deserves a dedicated visual instead of only a text statistic.

Current source: diagnostics described in docs/notes/research_progress.md and Track 1 visualization notebooks.

Ideal export: one side-by-side figure for a representative frame: raw point cloud, identity project→unproject, Stage0 recon, Stage1 recon.

5. Stage3 head sweep summary once runs settle

Why it matters: the current page is strongest on diagnosis. A final paper-grade version should also show the next acceptance metric once Stage3 results are stable enough to summarize.

Current gap: the repo has Stage3 code paths and launch artifacts, but not yet a clean final table that is safe to present as a stable outcome.

Suggested remote generation workflow

If you want to use the HPC environment later, the most useful remote task is not generic browsing. It is targeted export work: render notebook figures to PNG, export a short GIF, and save those assets into a public-facing folder that can be copied into docs/assets/.

Recommended output naming: docs/assets/track1-best-panel.png, docs/assets/track1-failure-panel.png, docs/assets/track2-banding-montage.png, docs/assets/track2-ri-comparison.gif, docs/assets/projection-retention-decomposition.png.
