\section{Introduction}\label{sec:introduction}
This thesis studies end-to-end LiDAR range-image compression with explicit ROI-aware bit allocation for downstream perception. The core question is whether adaptive latent quantization can preserve task-relevant regions while reducing bitrate. The current implementation uses a staged protocol:
\begin{itemize}
    \item \textbf{Stage0}: uniform quantization baseline (ROI-unaware).
    \item \textbf{Stage1}: adaptive quantization with ROI supervision, no teacher.
    \item \textbf{Stage2}: Stage1 plus teacher distillation.
    \item \textbf{Stage3}: Stage2 objective with stronger multi-scale ROI heads and detector-facing calibration protocol.
\end{itemize}

This document consolidates the actual code path, loss equations, experiment settings, and intermediate-to-latest results from the current repository and logs.

\section{Method and Stage Definitions}\label{sec:method}
\subsection{Input and model pipeline}
Each LiDAR scan is projected to a 5-channel range image $x \in \mathbb{R}^{B \times 5 \times H \times W}$ with channels \texttt{range, intensity, x, y, z}. The forward path is:
\begin{equation}
x \xrightarrow[]{\text{encoder}} z \xrightarrow[]{\text{importance head}} p \xrightarrow[]{\text{quantization}} \hat{z} \xrightarrow[]{\text{decoder}} \hat{x}.
\end{equation}
Backbones are ResNet-style encoder or DarkNet encoder, with latent projection to 64 channels. For Stage3 head variants, multi-stage encoder taps are additionally consumed by the ROI head.

\subsection{Adaptive quantization}
For adaptive mode, the importance map $p \in [0,1]$ controls quantization levels:
\begin{equation}
\ell = b_{\text{bg}} + p\,(b_{\text{roi}} - b_{\text{bg}}),
\end{equation}
followed by rounding/clamping $\tilde{\ell} = \max(2,\text{round}(\ell))$. With per-sample normalization:
\begin{equation}
z_{\text{norm}} = \frac{z - z_{\min}}{z_{\max}-z_{\min}+\epsilon}, \quad
q = \text{round}\!\left(z_{\text{norm}}(\tilde{\ell}-1)\right),
\end{equation}
\begin{equation}
\hat{z} = \frac{q}{\tilde{\ell}-1+\epsilon}(z_{\max}-z_{\min}) + z_{\min}.
\end{equation}
In this codebase, larger importance yields larger level count (finer quantization), not coarser.

\subsection{Stage definitions in this thesis}
\begin{itemize}
    \item \textbf{Stage0}: \texttt{quantizer\_mode=uniform}. No teacher, no importance supervision.
    \item \textbf{Stage1}: \texttt{quantizer\_mode=adaptive} + ROI supervision from SemanticKITTI ROI mask.
    \item \textbf{Stage2}: Stage1 + teacher distillation (\texttt{pointpillars\_zhulf} backend).
    \item \textbf{Stage3}: Stage2 objective maintained, but ROI head replaced by multi-scale variants (\texttt{bifpn, deformable\_msa, dynamic, rangeformer, frnet}).
\end{itemize}

\section{Loss Formulation}\label{sec:loss}
\subsection{Unified objective}
The implemented training objective is:
\begin{equation}
\mathcal{L}_{\text{total}} =
\lambda_{\text{recon}}\mathcal{L}_{\text{recon}} +
\lambda_{\text{distill}}\mathcal{L}_{\text{distill}} +
\lambda_{\text{rate}}\mathcal{L}_{\text{rate}} +
\lambda_{\text{imp}}\mathcal{L}_{\text{imp}} +
\lambda_{\text{sep}}\mathcal{L}_{\text{sep}}.
\label{eq:total-loss}
\end{equation}

\subsection{Reconstruction loss}
\begin{equation}
\mathcal{L}_{\text{recon}} = \text{MSE}(\hat{x}, x).
\end{equation}

\subsection{Rate proxy loss}
Let normalized level map be:
\begin{equation}
\ell_{\text{norm}} = \frac{\tilde{\ell} - b_{\text{bg}}}{b_{\text{roi}} - b_{\text{bg}}}.
\end{equation}
Three implemented modes are:
\begin{equation}
\mathcal{L}_{\text{rate}}^{\text{global}} = \text{mean}(\tilde{\ell}),
\end{equation}
\begin{equation}
\mathcal{L}_{\text{rate}}^{\text{norm-global}} = \text{mean}(\ell_{\text{norm}}),
\end{equation}
\begin{equation}
\mathcal{L}_{\text{rate}}^{\text{norm-bg}} =
\frac{\sum (\ell_{\text{norm}} \odot w_{\text{bg}})}{\sum w_{\text{bg}}},
\end{equation}
where
\begin{equation}
w_{\text{bg}}=
\begin{cases}
1-m, & \text{if ROI target } m \text{ exists},\\
1-\text{stopgrad}(p), & \text{otherwise}.
\end{cases}
\end{equation}

\subsection{Importance supervision loss}
\begin{equation}
\mathcal{L}_{\text{imp}} =
\text{BCEWithLogits}(s, m; \text{pos\_weight}),
\end{equation}
with logits $s$ from the importance head and ROI target $m$. In \texttt{weighted\_bce} mode:
\begin{equation}
\text{pos\_weight}=
\text{clip}\!\left(\frac{N_{\text{neg}}}{\max(N_{\text{pos}},1)},\,1,\,w_{\max}\right).
\end{equation}
ROI target downsampling mode is one of \texttt{nearest}, \texttt{maxpool}, \texttt{area}.

\subsection{ROI/BG separation loss}
\begin{equation}
\mathcal{L}_{\text{sep}} =
\max\!\left(0,\;\delta - (\mu_{\text{roi}} - \mu_{\text{bg}})\right),
\end{equation}
where $\mu_{\text{roi}}$ and $\mu_{\text{bg}}$ are mean sigmoid importance over ROI and background masks.

\subsection{Distillation loss}
The composite distillation loss is:
\begin{equation}
\mathcal{L}_{\text{distill}} =
\alpha_f \mathcal{L}_f + \alpha_\ell \mathcal{L}_\ell.
\end{equation}
Feature distillation uses weighted MSE between channel-mean student latent and channel-mean teacher feature. Logit distillation uses mode \texttt{auto/kl/bce/mse}; in \texttt{auto}, single-channel logits use BCE and multi-channel logits use KL.

\section{Experimental Setup}\label{sec:setup}
\subsection{Data and training protocol}
The training dataset is SemanticKITTI projected to range images, using sequences \texttt{00,01,02,03,04,05,06,07,09,10}. For fast diagnosis, a 16-frame evaluation subset is also used in notebook-based quick evaluation and oracle/native analysis.

\subsection{Model and optimization defaults}
\begin{itemize}
    \item Input channels: 5.
    \item Latent channels: 64.
    \item Backbones: ResNet encoder or DarkNet encoder.
    \item Adaptive quantization defaults: $b_{\text{roi}}=256$, $b_{\text{bg}}=16$.
    \item Uniform baseline bits: typically 4/6/8 in sweep, and 6/8 in fair-budget pilot.
    \item Optimizer: AdamW; default learning rate $10^{-4}$.
    \item Default ROI target mode in current recipes: \texttt{maxpool}.
\end{itemize}

\section{Results and Intermediate Findings}\label{sec:results}
\subsection{Cycle B failure signal: importance collapse}
Early experiments exposed a failure mode where importance predictions stayed near floor values, causing empty ROI prediction at threshold 0.5 in some runs. A representative notebook diagnosis reported:
\begin{itemize}
    \item $TP=0$, $FP=0$, $FN=70$ at threshold 0.5.
    \item importance range close to floor (example: min 0.0100, max 0.0307, mean 0.0127).
\end{itemize}
This motivated replacing legacy objective terms with normalized rate loss and weighted ROI supervision.

\subsection{Quick 16-frame analysis snapshot}
Quick evaluation over 16 frames showed that fixed-threshold ROI selectivity remained unstable. Example summary values:
\begin{itemize}
    \item Stage0 uniform baseline (ResNet q8): all-MSE 13.4235, ROI-MSE 24.8127.
    \item Stage1 adaptive (ResNet): all-MSE 10.9570, ROI-MSE 19.8815.
    \item Stage2 adaptive + distill (DarkNet, $\lambda_d=0.1$): all-MSE 29.5963, ROI-MSE 86.1911.
\end{itemize}
These values are directly from \texttt{notebooks/quick\_eval\_summary\_16frames.csv}. Across several runs, large false-positive behavior and threshold sensitivity were observed.

\section{Discussion}\label{sec:discussion}
\subsection{What improved}
The transition from legacy objective to balanced recipes, combined with stronger ROI head capacity and real teacher wiring, removed several early reproducibility and training-instability issues. Logging and ledger automation also enabled consistent cross-run analysis.

\subsection{What remains unresolved}
Two issues remain central:
\begin{itemize}
    \item importance calibration instability across runs (threshold-sensitive behavior),
    \item weak ROI selectivity at fixed threshold, often with high false positives.
\end{itemize}
Therefore, low final training loss alone cannot support deployment claims; detector-facing operating-point analysis is required.

\section{Conclusion}\label{sec:conclusion}
This thesis draft documents an implemented Stage0--Stage3 LiDAR compression pipeline with explicit, code-faithful objectives and experiment settings. Intermediate and latest results show that adaptive quantization has clear potential under reliable ROI signals, but current performance is constrained by importance-map calibration and selectivity. Ongoing Stage3 multi-scale head experiments are designed to directly address this bottleneck and complete detector-facing validation.
